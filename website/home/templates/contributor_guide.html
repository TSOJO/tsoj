Contributor Guide

Create problem

You can create a problem by pressing the green "Create" button in the problems page.

Problem ID

The problem ID should be unique. It should consists of the type of the problem and a number, seperated by a hyphen. For example, "Algo-1", "GCSE-8".
Common problem types include:

  • AL: A Level problems
  • AQAAsm: AQA Assembly problems
  • GCSE: GCSE problems
  • Intro: Introductory problems
  • Algo: Basic problems for algorithmic training
  • CP: Competitive programming problems
  • BIOxx: BIO (Round 1) problems at year 19xx or 20xx.

The number following the problem type should be one larger than the current maximum number among the problems of the same type.

Public problems

Non-public problems can only be seen by contributors and admins. Public problems can be seen by everyone in TSOJ.

Writing a problem description

The description will be rendered by Markdown and MathJax. Please see the Markdown syntax. To type in $\LaTeX$ math mode (inline), type them between two '$' symbols. For example, type $689 \times 777 = 535353$ for $689 \times 777 = 535353$. To type it in a new line, type them between two "$$". For instance, type $$\sum_{k=1}^{\infty} \frac{1}{k^2} = \frac{\pi^2}{6}$$ for $$\sum_{k=1}^{\infty} \frac{1}{k^2} = \frac{\pi^2}{6}$$

Input format

To improve consistency, the input for every testcase will be modified by TSOJ with the following:

  • Removing empty lines
  • Removing leading and trailing whitespaces in each line
  • Adding a trailing line break(\n), if it does not already exist
Generate answers

To save time, this function can be used to automatically generate the answers of each testcase, using the given inputs and program. Please be aware that pressing the blue "Generate answers" button will overwrite the answers in every testcase.

Grader

Grader can be used if a problem accepts multiple outputs. The grader program will be executed with each testcase. The input for this program consists of the original input and the user output of the testcase, seperated by a line break. The grader should output "AC" or "WA" (without quotes) in one line. For example, this grader program accepts any number that is a multiple of a given $k$.

k = int(input()) # get input input() # get the empty line try: x = int(input()) # get user output # grade the output if x % k == 0: print('AC') else: print('WA') except ValueError: print('WA') # the user output is not a number
Important note: any error in the grader at all will be reported to the user as 'System error'. For example, if there were no try/except clause in the code above, the grader would raise a ValueError if the user output were not a number.

Restricting languages

If the "Restrict problem to certain languages" checkbox is not checked, this problem can accept all languages as a submission. Otherwise, only the selected language(s) are accepted.
